## 变更清单

### 1) 失败分级（事项1）

- `NodePoller` 新增 `RegistrationOutcome`，返回 `success/retryable/code/message`
- 注册接口状态映射：
  - 200/201 -> 成功
  - 401/403/409/422 -> 不可重试
  - 429/5xx/网络异常 -> 可重试
- `certman-agent` 启动行为：
  - 注册失败时输出 `register_status=failed register_code=... retryable=...`
  - 不可重试退出码 `2`
  - 可重试退出码 `3`

涉及文件：
- `certman/node_agent/poller.py`
- `certman/node_agent/agent.py`

### 2) 授权联动（事项2）

- node-agent 路由增强：
  - `disabled` 节点返回 `403 AUTH_NODE_DISABLED`
  - `pending`/未审批返回 `401 AUTH_NODE_NOT_APPROVED`
  - poll/result 成功后更新 `node.last_seen`
- 保持 nonce 重放保护与 job-node 绑定校验

涉及文件：
- `certman/api/routes/node_agent.py`

### 3) 部署模板（事项3）

- K8s：模板新增
  - `config-agent.toml`（agent 运行配置）
  - `Secret certman-agent-registration`（registration-token）
  - `Deployment certman-agent`
- 新增 agent compose 示例配置

涉及文件：
- `docker-compose.yml`
